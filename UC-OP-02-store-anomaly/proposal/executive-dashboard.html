Key Finding: 51% Revenue Decline at 8th & Broadway — Detected 4-6 Weeks Late
Store US00001 experienced a sustained revenue decline from July 2025 through January 2026, losing 51% of weekly revenue. The decline was only discovered during a manual quarterly review in January 2026. SPC-based monitoring would have flagged the anomaly by mid-November 2025, enabling 4-6 weeks of earlier intervention and an estimated $18K-$30K in recovered revenue.
Problem & Solution
The Problem

Luckin Coffee US currently relies on manual periodic reviews to detect store performance anomalies. With 8+ stores operating, quarterly or monthly reviews mean significant revenue can be lost before issues are identified. The 8th & Broadway case demonstrates that a 51% decline went unnoticed for 10+ weeks, resulting in substantial financial impact and delayed corrective action.

The Solution

Implement Statistical Process Control (SPC) with Western Electric rules for continuous automated monitoring. The system computes rolling means, standard deviations, and applies 8 WE rules to detect anomalies in real-time. A composite health score across 5 dimensions (revenue, operations, quality, staffing, customer) provides at-a-glance portfolio status with automated alerting via Grafana.

Deployment Note
The entire pipeline is designed to run on existing infrastructure with minimal footprint. No additional servers required — the Python orchestrator runs as a cron job on the existing application server. All SQL scripts are idempotent and safe for repeated execution. Grafana dashboards connect directly to the MySQL analytics schema via read-only credentials.
